# SecureJournal v0.8.9

**Release Date:** 2026-03-02

## Summary
This release fully removes all in-memory caching for user, group, and project access. All authorization and data reads are now store-backed, ensuring consistent and up-to-date access control. SQLite shared cache and pooling are disabled for maximum data consistency. The regression test suite has been updated and all tests pass (31/31).

## Highlights
- Full removal of all in-memory caching for user, group, and project access
- All authorization and data reads are now store-backed
- SQLite shared cache and pooling disabled
- Added RefreshSharedStateFromStore() to SecureJournalAppService and wired into all entry points
- Regression test suite updated and all tests pass
- No compile or lint errors in core service or data access layers

## Bug Fixes
- Fixed stale project access after group assignment by removing cached authorization data
- Ensured dashboard, journal, and admin operations always reflect current database state

## Security
- Improved consistency and reliability of access control logic
- Eliminated subtle stale-read bugs in multi-instance scenarios

## Upgrade Notes
- No breaking changes to public APIs
- All users and admins will see immediate updates to project/group access after changes
